Project run notes

This project uses:
- frontend: `front`
- backend: `our_project`
- database: PostgreSQL on `127.0.0.1:5432`

Recommended startup order
1. Start PostgreSQL
2. Start backend
3. Start frontend

Recommended shutdown order
1. Stop frontend
2. Stop backend
3. Stop PostgreSQL

Database: start and connect

In cmd:
sc start postgresql-x64-18
"C:\Program Files\PostgreSQL\18\bin\psql.exe" -h 127.0.0.1 -p 5432 -U db -d tongwen

In PowerShell:
sc.exe start postgresql-x64-18
& "C:\Program Files\PostgreSQL\18\bin\psql.exe" -h 127.0.0.1 -p 5432 -U db -d tongwen

Exit psql:
\q

Stop database

In cmd:
sc stop postgresql-x64-18

In PowerShell:
Stop-Service -Name "postgresql-x64-18"

Backend: start with in-memory mode

In cmd:
cd /d C:\Users\Jimmy\projects\front_backend\our_project
sbt run

In PowerShell:
Set-Location C:\Users\Jimmy\projects\front_backend\our_project
sbt run

Backend: start with PostgreSQL mode

The backend chooses PostgreSQL if:
- `RIICHI_STORAGE=postgres`, or
- `RIICHI_DB_URL` is set, or
- template-style `DB_*` variables are present and normalized into postgres settings

Direct URL style in cmd:
cd /d C:\Users\Jimmy\projects\front_backend\our_project
set RIICHI_STORAGE=postgres
set RIICHI_DB_URL=jdbc:postgresql://127.0.0.1:5432/tongwen
set RIICHI_DB_USER=db
set RIICHI_DB_PASSWORD=root
sbt run

Template-style variables in cmd:
cd /d C:\Users\Jimmy\projects\front_backend\our_project
set RIICHI_STORAGE=postgres
set DB_HOST=127.0.0.1
set DB_PORT=5432
set DB_NAME=tongwen
set DB_USER=db
set DB_PASSWORD=root
sbt run

Direct URL style in PowerShell:
Set-Location C:\Users\Jimmy\projects\front_backend\our_project
$env:RIICHI_STORAGE="postgres"
$env:RIICHI_DB_URL="jdbc:postgresql://127.0.0.1:5432/tongwen"
$env:RIICHI_DB_USER="db"
$env:RIICHI_DB_PASSWORD="root"
sbt run

Template-style variables in PowerShell:
Set-Location C:\Users\Jimmy\projects\front_backend\our_project
$env:RIICHI_STORAGE="postgres"
$env:DB_HOST="127.0.0.1"
$env:DB_PORT="5432"
$env:DB_NAME="tongwen"
$env:DB_USER="db"
$env:DB_PASSWORD="root"
sbt run

How to confirm backend mode
- Check backend startup output
- In postgres mode, the backend should report PostgreSQL-backed startup rather than the default in-memory path
- If it still starts in memory mode, restart from the same terminal after setting the environment variables again

Frontend: start

In cmd:
cd /d C:\Users\Jimmy\projects\front_backend\front
npm run dev

In PowerShell:
Set-Location C:\Users\Jimmy\projects\front_backend\front
npm run dev

Frontend API proxy
- frontend `/api` requests are proxied to `http://127.0.0.1:8080`
- backend should be running on port `8080` before frontend testing

Useful checks

Check PostgreSQL service status in cmd:
sc query postgresql-x64-18

Check current env vars in cmd before starting backend:
set RIICHI_
set DB_

Check database health from backend project:
cd /d C:\Users\Jimmy\projects\front_backend\our_project
sbt "runMain riichinexus.riichiNexusDbAdmin health"

Additional admin checks:
sbt "runMain riichinexus.riichiNexusDbAdmin stats"

Notes
- the backend main entry is `our_project/src/main/scala/Main.scala`
- database assembly now lives under `riichinexus.bootstrap`
- postgres and memory implementations now live under `riichinexus.infrastructure.*`
- if the database password changes, update the startup commands above
- if `psql` is not in PATH, use:
  `C:\Program Files\PostgreSQL\18\bin\psql.exe`
